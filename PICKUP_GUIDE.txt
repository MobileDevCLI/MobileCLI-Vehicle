1. FULL REVERSE ENGINEERING of head unit MCU serial protocol:
   - /dev/ttyHS1 @ 115200 baud
   - Frame format: 0D 0A [LEN] [DATA...LEN bytes]
   - Command types: 0x71=heartbeat, 0x8E=IMU, 0xA5=CAN, 0xA6=ATA
   - Checksum: ~(sum of LEN + DATA) & 0xFF
   - Send: 0D 0A [LEN] [DATA] [~CHK] 00

2. FULL REVERSE ENGINEERING of CAN bus data flow:
   - EventCenter APK decompiled at ~/tmp/eventcenter_dex/
   - canbus2 APK decompiled at ~/tmp/smali_out/
   - Data flow: ttyHS1 → SerialReadThread → EventService → CanbusDataService
   - LuZheng parser: speed(0x16), RPM(0x50), doors(0x24), AC(0x28)

3. HARDWARE FINDINGS:
   - Hardwired CAN adapter has ONLY 2 PINS (power/ground) — NO CAN DATA
   - OBD2 port has CAN-H on pin 6, CAN-L on pin 14 — but adapter doesn't use them
   - Head unit supports 22 CAN box brands (see list below)
   - Current config: LuZheng brand (#3), Toyota brand=1, carType=0x22E (FJ)

OPTION A: WiFi ELM327 (BEST — EASIEST PATH)
---------------------------------------------
WiFi ELM327 adapters create a TCP server, typically at 192.168.0.10:35000.
Python connects directly — no pairing, no kernel issues, no APK needed.

Steps:
1. Plug WiFi ELM327 into OBD2 port under dash (left of steering wheel)
2. Turn ignition ON
3. Connect head unit WiFi to the ELM327's hotspot (usually "WiFi_OBDII" or similar)
4. Update server.py to add WiFi ELM327 support:

   import socket
   elm_sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
   elm_sock.connect(('192.168.0.10', 35000))  # or 192.168.1.10:35000

   def elm_cmd(cmd, timeout=3):
       elm_sock.send((cmd + '\r').encode())
       time.sleep(timeout)
       return elm_sock.recv(4096).decode()

   # Initialize:
   elm_cmd('ATZ', 2)    # Reset
   elm_cmd('ATE0', 1)   # Echo off
   elm_cmd('ATL0', 1)   # Linefeed off
   elm_cmd('ATS0', 1)   # Spaces off
   elm_cmd('ATSP0', 1)  # Auto protocol
   elm_cmd('ATST32', 1) # Timeout

   # Read PIDs:
   rpm_resp = elm_cmd('010C', 2)    # RPM
   speed_resp = elm_cmd('010D', 2)  # Speed
   coolant_resp = elm_cmd('0105', 2) # Coolant temp
   throttle_resp = elm_cmd('0111', 2) # Throttle
   dtc_resp = elm_cmd('03', 3)      # Read DTCs

5. Integrate into web dashboard at localhost:8099


OPTION B: Bluetooth ELM327
---------------------------------------------
Steps:
1. Plug BT ELM327 into OBD2 port, turn ignition ON
2. Open BT settings: am start -n com.szchoiceway.btsuite/.BTMainActivity
3. Pair the OBD2 device (usually named "OBD" or "OBDII" or "V-LINK", PIN: 1234)
4. Check it's paired: termux-bluetooth-paired
5. Install the APK (user must do from file manager):
   - Open file manager on head unit
   - Navigate to /sdcard/Download/OBD2Scanner.apk
   - Tap to install
6. Launch: am start -n com.mobilecli.obd2/.OBD2Scanner
7. Tap SCAN BT → tap the OBD device → tap SCAN OBD2

If APK install doesn't work, rebuild:
   cd ~/obd2-scanner && bash build.sh

================================================================================
  OBD2 PID REFERENCE (FOR ELM327)
================================================================================

Mode 01 (Current Data):
  0100 — Supported PIDs [01-20]
  0104 — Engine Load (%)
  0105 — Coolant Temperature (C)
  010C — Engine RPM
  010D — Vehicle Speed (km/h)
  010F — Intake Air Temperature (C)
  0111 — Throttle Position (%)
  012F — Fuel Level (%)
  0142 — Control Module Voltage (V)

Mode 03: Read DTCs
Mode 04: Clear DTCs
Mode 09: Vehicle Info (VIN, ECU name)
  0902 — VIN

Parsing:
  RPM = ((A * 256) + B) / 4
  Speed = A (km/h)
  Coolant = A - 40 (C)
  Throttle = A * 100 / 255 (%)
  Engine Load = A * 100 / 255 (%)
  Intake Temp = A - 40 (C)

DTCs (Mode 03 response "43"):
  Each DTC = 2 bytes: [high][low]
  Prefix: bits 7-6 of high byte: 0=P, 1=C, 2=B, 3=U
  Code: remaining 14 bits as hex
